Here’s how to turn your HTML and CSS into a live, free website using GitHub Pages:

1. Prepare your files

Make sure you have at least these in a folder on your computer:

pgsql
/SmartMeal-Support
├── index.html
├── styles.css
└── logo.png         ← (copy your logo here)

Your index.html already links to styles.css and logo.png, so just keep them together.

2. Create a GitHub repository

  1. Sign in (or sign up) at https://github.com.

  2. Click New repository.

  3. Set the Repository name to:

    • username.github.io
      (replace username with your GitHub username)

  4. Optionally add a Description (e.g. “SmartMeal Support Center”).

  5. Leave it Public and do not initialize with a README (you’ll upload your own files).

  6. Click Create repository.

3. Upload your site

You have two choices:

A) Via the GitHub web interface

  1. In your new repo, click Upload files.

  2. Drag & drop your index.html, styles.css, and logo.png.

  3. Commit the changes at the bottom (e.g. “Add initial site files”).

B) Via Git on your computer

bash
# In your terminal, from your project folder:
cd /path/to/SmartMeal-Support
git init
git remote add origin https://github.com/username/username.github.io.git
git add .
git commit -m "Initial site upload"
git push -u origin master

(Replace username and branch name if needed.)

4. Enable GitHub Pages

  1. Go to your repo’s Settings → Pages.

  2. Under “Build and deployment,” set Branch to master (or main).

  3. Click Save.

GitHub will show a URL like

arduino
https://username.github.io/

Your site is live there within a minute or two!

5. Verify & troubleshoot

  • Visit https://username.github.io/ in your browser.

  • If you see your SmartMeal Support page styled correctly, you’re done!

  • If you get a 404 or old content:

    • Make sure your files are at the root of the branch you chose.

    • Check Settings → Pages shows the correct branch/folder.

    • Clear your cache or try an incognito window.

6. (Optional) Custom domain

If you have your own domain (e.g. support.smartmeal.com):

  1. In your Pages settings, enter your custom domain.

  2. On your DNS provider, add a CNAME record pointing your domain to

    lua
    username.github.io

  3. GitHub will provision HTTPS automatically once DNS has propagated.
